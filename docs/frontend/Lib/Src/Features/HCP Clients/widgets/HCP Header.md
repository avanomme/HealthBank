# HcpHeader

File: `frontend/lib/src/features/hcp_clients/widgets/hcp_header.dart`

## Overview

`HcpHeader` is a specialized header widget used throughout Healthcare Professional (HCP) pages. It wraps the shared `Header` component and preconfigures navigation items specific to the HCP role.

The widget provides navigation links to:

- Dashboard
- Clients
- Reports

It also supports user interactions such as notifications and profile actions while maintaining consistent navigation behavior across HCP pages.

`HcpHeader` implements `PreferredSizeWidget`, allowing it to be used directly in scaffold-based layouts that expect an app bar–like header component.

## Architecture / Design

`HcpHeader` is designed as a thin wrapper around the shared `Header` widget located in the core UI layer.

### Core Responsibilities

- Configure HCP-specific navigation items.
- Provide routing behavior for navigation interactions.
- Pass optional UI actions (notifications, profile menu).
- Maintain consistent navigation structure for all HCP pages.

### Header Composition

The widget delegates most functionality to the base `Header` component while injecting HCP-specific configuration:

- Navigation items are generated by `_getNavItems()`.
- Active route highlighting is handled through `currentRoute`.
- Navigation actions trigger route transitions using `GoRouter`.

### Navigation Items

The navigation items are localized and constructed dynamically using `context.l10n`.

Defined routes:

- `/hcp/dashboard`
- `/hcp/clients`
- `/hcp/reports`

Each item is represented by a `NavItem` object used by the base `Header`.

### Navigation Behavior

Navigation is handled through the `go_router` package.

- Selecting a navigation item calls `context.go(route)`.
- Clicking the application logo redirects the user to the HCP dashboard.
